# LOOM Preview 生成提示词

你正在为 LOOM 项目生成 HTML 预览。下面是项目数据和使用要求。

## HTML 的优势（你要充分利用）

HTML 相比 Markdown 的核心优势，你的预览必须体现：

1. **信息密度** — 用 SVG 画依赖图（不是 ASCII），用 CSS 做颜色编码，用表格展示结构化数据
2. **视觉清晰** — 用 tabs 组织不同板块（哲学/愿景/架构/Intent Map/进度），不要一长串滚动
3. **双向交互** — Intent 可点击展开详情，有"copy as prompt"按钮导出修改建议
4. **响应式** — 手机也能看，布局自适应

## 数据

下面是项目的完整数据（JSON 格式）：

```json
{{LOOM_DATA}}
```

## 你要生成的 HTML

生成一个**单文件 HTML**（所有 CSS/JS 内联，零外部依赖），包含以下板块：

### 1. 顶部：项目概览

- 版本号 badge
- 进度条（completed/total，带百分比）
- 统计卡片：总数 / 已完成 / 进行中 / 待执行 / 阻塞（用颜色区分）

### 2. Intent Map 可视化

- **SVG 依赖图**：节点是 Intent，边是 depends_on 关系。节点颜色按状态编码：
  - pending = 灰色
  - in_progress = 黄色
  - completed = 绿色
  - blocked = 红色
  - needs_review = 蓝色
- 节点可点击，点击后右侧面板显示该 Intent 的详情（叙事、验收契约、哲学锚点）
- 下方表格列出所有 Intent（ID / 状态徽章 / 依赖 / 验收契约摘要）

### 3. 哲学文档（tabs 切换）

- 每个 .md 文件一个 tab
- 内容渲染：标题层级、粗体、列表、引用、代码块
- 底线内化声明的 checkbox 显示勾选状态

### 4. 愿景

- 北极星高亮显示
- 意图叙事列表，每个叙事可展开/折叠

### 5. 架构（如果有）

- 系统设计内容渲染

### 6. 底部：操作区

- "Copy 修改建议" 按钮 — 把当前项目状态复制为一段自然语言，用户可以粘贴给 Agent 说"请调整 XXX"
- "重新生成预览" 提示 — 提醒用户改完 md 后跑 `loom preview` 重新生成

## 设计要求

- **暗色主题**：背景 #0d1117，卡片 #161b22，边框 #30363d，文字 #c9d1d9
- **强调色**：蓝色 #58a6ff（链接/标题），绿色 #3fb950（完成），黄色 #d29922（进行中），红色 #f85149（阻塞）
- **字体**：系统字体栈，代码用 monospace
- **零依赖**：不用任何 CDN、外部库。纯 HTML + CSS + JS
- **单文件**：所有内容内联，双击即可在浏览器打开

## 输出

把生成的 HTML 写到 `loom-preview.html`。完成后告诉用户文件路径。
