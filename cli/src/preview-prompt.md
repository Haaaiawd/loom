# LOOM Preview 生成提示词

你的任务：读 `.loom/` 目录下当前版本的所有文件，把信息**重新组织**成一份人类最容易理解的 HTML 页面，写到 `loom-preview.html`。

## 核心原则：重组，不是搬运

人类读 md 里的大段文字很累。你不是把 md 渲染成 HTML，你是把信息**提炼、拆解、重组**，用 HTML 的视觉和交互优势让它在人类直觉里更容易消化。

举例：
- 哲学文档里 500 字的"核心信念"段落 → 提炼成 3 张卡片，每张一个关键句
- Intent Map 的 JSON 依赖关系 → 画成 SVG 依赖图，一眼看出谁阻塞谁
- 愿景里散落的意图叙事 → 每个意图一张可展开卡片，折叠时只显示标题
- 验证记录的 JSON → 时间线展示，通过/偏离用颜色区分

## 去读哪些文件

## 一、什么是"符合人类直觉"

人类看页面的方式是**扫描**，不是阅读。先看大字、颜色、形状，决定哪里值得停，才会读文字。你的 HTML 要顺应这个认知路径：

### 视觉权重

最重要的信息最大、最醒目。次要信息小、灰、退后。具体规则：

- **北极星**（一句话定义产品为什么存在）→ 全页最大字号，最醒目位置
- **进度数字**（完成多少）→ 大数字 + 颜色编码
- **Intent ID** → 中等字号，等宽字体
- **验收契约正文** → 小字，可展开才显示
- **辅助说明** → 最小字，灰色

不要所有文字一样大。人类靠大小判断重要性。

### 渐进披露

不要一次把所有信息摊开。先给摘要，想看详情再展开：

- Intent 卡片默认折叠，只显示 ID + 状态 + 一句话
- 点击展开后显示：意图叙事、验收契约、哲学锚点、验证历史
- 哲学的"决策原则"默认只显示标题，点击展开具体原则

### 信息分组

相关信息放一起，不相关的用空间分隔。人类靠距离判断"这两个东西有没有关系"。

- 进度概览自成一个区块
- Intent 依赖图 + 详情面板自成一个区块
- 哲学、愿景、架构各自独立区块
- 区块之间用足够留白分隔，不要挤

### 颜色编码

用颜色传达状态，但**不要只靠颜色**（色盲友好）：

- 完成 = 绿色 + ✓ 标记
- 进行中 = 黄色 + ◐ 标记
- 待执行 = 灰色 + ○ 标记
- 阻塞 = 红色 + ✕ 标记
- 需复审 = 蓝色 + ? 标记

## 二、交互设计

### Tab 切换

用 tab 组织大板块（Intent Map / 哲学 / 愿景 / 架构）。

实现要点：
- 所有 tab 内容先渲染出来（堆叠），再用 JS 切换显示
- 选中的 tab 项加视觉标记（底部边框或背景色）
- 切换时内容区淡入，不要硬切
- 默认显示第一个 tab

### 点击展开

Intent 卡片、哲学章节、愿景叙事都用可折叠面板。

实现要点：
- 用 `max-height` 过渡，不要 `display: none`（后者无法动画）
- 折叠时 `max-height: 0; overflow: hidden`
- 展开时 `max-height: 2000px`（足够大）
- `transition: max-height 0.3s ease`
- 折叠状态有视觉提示（▶ / ▼ 或 +/−）

### SVG 节点交互

Intent 依赖图的节点可点击：

- 点击节点 → 高亮该节点（加描边）+ 更新详情面板
- 其他节点取消高亮
- 详情面板内容用 `innerHTML` 更新
- 详情面板如果有滚动条，滚动到顶部

### 过滤/排序（如果 Intent 多）

Intent 超过 10 个时，加过滤和排序：
- 按状态过滤（全部 / 待执行 / 进行中 / 完成 / 阻塞）
- 按依赖数排序
- 输入框模糊搜索

## 三、视觉设计原则

参考 Anthropic 和 CopilotKit 的 Generative UI 设计系统：

### 平面克制

- **无渐变**：纯色背景，不要 linear-gradient
- **无阴影**：不要 box-shadow，用边框区分层次
- **无模糊**：不要 filter: blur
- **无发光**：不要 text-shadow 或 glow 效果
- **无装饰性 emoji**：用 CSS 形状或 SVG 路径做图标

### 字体克制

- 只用两种字重：400（正文）和 500（标题/强调）
- 不要 600、700——太重，破坏平面感
- 标题不要全大写，不要 Title Case，用句首大写
- 正文不要句中加粗，用 `code style` 标记实体名
- 最小字号 12px，再小就读不清

### 边框和间距

- 边框 `1px solid`（或 `0.5px` 如果支持）
- 卡片圆角 8-12px
- 卡片内边距 16-20px
- 卡片之间间距 16-24px
- 区块之间间距 32-48px——大间距区分大板块

### CSS 变量

用 CSS 变量定义颜色，方便统一调整和适配明暗模式：

```css
:root {
  --bg: ...;
  --surface: ...;
  --border: ...;
  --text: ...;
  --text-muted: ...;
  --accent: ...;
  --success: ...;
  --warning: ...;
  --danger: ...;
}
```

不要在 HTML 里硬编码颜色。所有颜色引用变量。

### 配色自定

不要求暗色也不要求亮色。选你觉得让信息最易读的方案。关键是：
- 背景和文字对比度足够（WCAG AA 标准）
- 状态颜色区分明显
- 不要花哨——信息是主角，视觉是配角

## 四、信息组织方案

以下是重组方向，不是死规定——根据实际内容调整。

### 顶部：项目概览

一眼看到全局：
- 项目名 + 版本号
- 进度（完成/进行中/待执行/阻塞各多少，用大数字 + 颜色）
- 一句话说明项目在做什么（从哲学的北极星或愿景提取）

### Intent Map（核心区域）

人类最关心"项目推进到哪了"：

1. **SVG 依赖图**：节点是 Intent，边是 depends_on。拓扑排序布局，无依赖的在左，依赖链向右。节点颜色按状态编码。节点可点击
2. **Intent 卡片列表**：每个 Intent 一张卡片，默认折叠，显示 ID + 状态 + 一句话（从叙事提取）。点击展开显示完整详情
3. **详情内容**：意图叙事（为什么存在）、验收契约（什么算完成）、哲学锚点（受什么约束）、验证历史（通过/偏离/证据）

如果 Intent Map 还是模板，显示提示："等待 Architect 设计"。

### 哲学（重组，不是搬运）

不要把整个 md 渲染出来。提炼成视觉块：

- **北极星**：一句话，最醒目
- **核心信念**：每条一个卡片或一行
- **不可妥协的价值**：列表，每条附理由
- **反模式清单**：用视觉上明显"禁止"的样式（红色边框、✕ 标记）
- **决策原则**：遇到冲突时怎么取舍

md 里写得长的部分，提炼关键句。人类要的是"这个产品信什么"，不是全文。

### 愿景（重组）

- **北极星**：和哲学的北极星呼应，突出显示
- **问题空间**：一段话概括用户在什么场景下遇到什么痛苦
- **意图叙事**：每个意图一张卡片，标题是意图名，内容是"为什么存在"
- **不做什么**：和"做什么"同样醒目，每条附理由

### 架构（如果有）

- 系统设计概览
- 关键决策和 trade-offs

### 验证历史（如果有）

- 哪些 Intent 通过了
- 哪些偏离了，偏离证据是什么
- 时间线或列表展示

### 底部

- 提醒：这是只读投影，修改请编辑源文件后运行 `loom preview` 重新生成
- 可选：一个"复制项目状态"按钮，把当前进度复制为一段自然语言，方便用户粘贴给 Agent

## 五、技术约束

- **单文件**：所有 HTML/CSS/JS 内联，零外部依赖
- **零构建**：双击即可在浏览器打开
- **内联 SVG**：依赖图直接写 `<svg>`，不用 `<img>`
- **语义化 HTML**：用 `<section>` `<nav>` `<article>` `<button>`
- **响应式**：手机宽度下布局不崩（`@media (max-width: 768px)`）

## 六、质量检查清单

生成前逐项确认：

**功能**
- [ ] 双击能在浏览器打开，不需要任何外部资源
- [ ] 所有交互控件有事件处理
- [ ] SVG 依赖图节点颜色按状态编码
- [ ] 点击 Intent 能展开详情
- [ ] Tab 切换正常

**视觉**
- [ ] 无渐变、无阴影、无模糊、无发光
- [ ] 字重只有 400 和 500
- [ ] 所有颜色用 CSS 变量
- [ ] 边框统一 1px
- [ ] 信息层次清晰（大字重要，小字次要）

**内容**
- [ ] 信息是重组过的，不是 md 内容的简单渲染
- [ ] 哲学部分是提炼的，不是全文搬运
- [ ] 模板文件有提示标记
- [ ] 视觉本身自解释，不依赖外部说明

**可访问性**
- [ ] `@media (prefers-reduced-motion: reduce)` 禁用动画
- [ ] 对比度足够（暗背景用亮文字，反之）
- [ ] 点击区域 ≥ 44px
- [ ] 不仅靠颜色传达信息（颜色 + 图标 + 文字）

**响应式**
- [ ] 手机宽度下统计卡片改为网格
- [ ] SVG 依赖图横向滚动
- [ ] Tab 导航横向滚动
