# LOOM Preview 生成提示词

你的任务：为这个 LOOM 项目生成一份 HTML 可视化预览，写到 `loom-preview.html`。

## 你要做什么

读 `.loom/` 目录下当前版本的所有文件，把里面的信息**重新组织**成一份人类最容易理解的 HTML 页面。

注意：不是把 md 内容渲染成 HTML，是**重新组织**。人类读 md 里的大段文字很累，你要把信息提炼、拆解、重组，用 HTML 的优势让它在视觉上更容易消化。

## 为什么用 HTML 而不是直接给 md

参考 Anthropic Claude Code 团队的实践：超过 100 行的 Markdown 就没人读了。HTML 能做到 md 做不到的事——SVG 依赖图、可折叠卡片、点击展开、颜色编码、空间布局。这份预览是人类理解项目全局的唯一窗口，要让人**愿意打开、打开就能看懂、看懂就能决策**。

## 去读哪些文件

当前版本目录在 `.loom/current` 指针里（通常是 `.loom/v1/`）。读以下文件：

| 文件 | 内容 |
|---|---|
| `00_PHILOSOPHY/*.md` | 产品哲学（北极星、核心信念、反模式、决策原则） |
| `01_VISION.md` | 愿景（北极星、问题空间、意图清单、不做什么） |
| `02_ARCHITECTURE.md` | 架构（如果有） |
| `04_INTENT_MAP.json` | Intent Map（依赖关系、状态、验收契约） |
| `verifications/*.json` | 验证记录（如果有） |

如果文件还是模板（含 `<!-- LOOM_TEMPLATE -->` 标记），在对应区域显示提示："这是模板，等待 {角色} 填充"。

## 怎么组织信息

以下是信息重组的方向，不是死规定——根据实际内容调整，目标是**人类最易读**。

### 项目概览（顶部）

一眼看到项目全局状态：
- 版本号
- 进度（完成多少、进行多少、阻塞多少）
- 一句话说明项目在做什么（从哲学的北极星或愿景提取）

### Intent 进度（核心区域）

这是人类最关心的——项目推进到哪了。

- **SVG 依赖图**：节点是 Intent，边是 depends_on。节点颜色按状态区分（完成/进行中/待执行/阻塞）。拓扑排序布局，无依赖的在左，依赖链向右延伸
- **Intent 卡片列表**：每个 Intent 一张卡片，显示 ID、状态、一句话说明（从叙事提取）、验收契约摘要
- 点击 SVG 节点或卡片，展开该 Intent 的详情：意图叙事（为什么存在）、验收契约（什么算完成）、哲学锚点（受什么约束）、验证历史（通过/偏离/证据）

### 哲学（重组，不是搬运）

不要把整个 md 渲染出来。提炼成几个视觉块：

- **北极星**：一句话，最醒目的位置
- **核心信念**：每条一个卡片或一行
- **不可妥协的价值**：列表形式，每条附理由
- **反模式清单**：用视觉上明显"禁止"的样式（如红色边框、⚠️ 标记）
- **决策原则**：遇到冲突时怎么取舍

如果某个部分在 md 里写得很长，提炼关键句就行。人类要的是"这个产品信什么"，不是全文。

### 愿景（重组）

- **北极星**：和哲学的北极星呼应，突出显示
- **问题空间**：用一段话概括用户在什么场景下遇到什么痛苦
- **意图叙事**：每个意图一张卡片，标题是意图名，内容是"为什么存在"的叙事（不是功能描述）
- **不做什么**：用和"做什么"同样醒目的方式展示，每条附理由

### 架构（如果有）

- 系统设计概览
- 关键决策和 trade-offs

### 验证历史（如果有）

- 哪些 Intent 通过了验证
- 哪些偏离了，偏离的证据是什么
- 用时间线或列表展示

### 底部

- 提醒：这是只读投影，修改请编辑源文件后运行 `loom preview` 重新生成
- 可选：一个"复制项目状态"按钮，把当前进度复制为一段自然语言，方便用户粘贴给 Agent

## HTML 的优势——你要用到的

- **SVG**：依赖图直接用 `<svg>` 画，不要用 ASCII art
- **交互**：点击展开、折叠面板、tab 切换
- **颜色编码**：状态用颜色区分，反模式用警告色
- **空间布局**：用网格、卡片、间距组织信息，不要一长串滚动
- **响应式**：手机也能看

## 技术约束

- **单文件**：所有 HTML/CSS/JS 内联，零外部依赖
- **零构建**：双击即可在浏览器打开
- **风格自定**：选择适合阅读的视觉风格。不要求暗色也不要求亮色——你觉得哪种让信息更易读就用哪种。关键是层次清晰、信息不挤、重点突出

## 质量检查

生成前确认：

- [ ] 双击能在浏览器打开，不需要任何外部资源
- [ ] 信息是重组过的，不是 md 内容的简单渲染
- [ ] SVG 依赖图能画出来，节点颜色按状态编码
- [ ] 点击 Intent 能展开详情
- [ ] 哲学部分是提炼的，不是全文搬运
- [ ] 模板文件有提示标记
- [ ] 没有未闭合的标签或 JS 错误

## 输出

把 HTML 写到 `loom-preview.html`。完成后告诉用户文件路径。
