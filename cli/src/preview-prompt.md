# LOOM Preview 生成提示词

你的任务：读 `.loom/` 目录下当前版本的所有文件，把信息**拆解成关键要点**，用各种视觉形式呈现成一份 HTML 页面，写到 `loom-preview.html`。

## 核心原则：这是 PPT，不是书

人类看这份页面就像看 PPT——每屏一个要点，扫一眼就懂，不读全文。你的工作不是"把 md 内容展示出来"，是"把信息拆解成人类需要知道的关键点，用最合适的视觉形式呈现"。

### 书 vs PPT 的区别

| 书（错误做法） | PPT（正确做法） |
|---|---|
| 哲学全文渲染 | 北极星一句话 + 3 张信念卡片 |
| 愿景全文展示 | 问题空间一段话 + 意图叙事卡片 |
| Intent Map 表格 | SVG 依赖图 + 状态仪表盘 |
| 验证记录 JSON 列表 | 时间轴，通过/偏离一目了然 |
| 架构文档全文 | 关键决策对比表 |

### 拆解的粒度

每个信息块应该能回答一个问题：
- 这个产品为什么存在？→ 北极星一句话
- 产品信什么？→ 3-5 张信念卡片
- 什么绝对不做？→ 反模式清单（视觉警告）
- 项目推进到哪了？→ 进度仪表盘
- 谁阻塞谁？→ SVG 依赖图
- 这个 Intent 为什么存在？→ 一句话叙事
- 验证历史怎么样？→ 时间轴

如果一个块需要读超过 3 行文字才能理解，它太长了。

## 一、视觉形式库——用什么呈现什么

不同信息适合不同视觉形式。不要什么都用卡片或表格——选最合适的形式。

### 仪表盘（Dashboard）

**适合**：项目概览、进度状态

用大数字 + 颜色编码呈现关键指标：
- 完成 X / 总数 Y（大数字 + 百分比）
- 进行中 N（黄色数字）
- 阻塞 M（红色数字）

数字要大（24-32px），标签要小（12-13px）。一眼看到状态。

### SVG 依赖图

**适合**：Intent 之间的依赖关系

节点是 Intent，边是 depends_on。拓扑排序布局，无依赖的在左，依赖链向右。节点颜色按状态编码。节点可点击，点击后侧边面板显示详情。

### 时间轴（Timeline）

**适合**：验证历史、项目演进

横向或纵向时间轴，每个节点是一个事件（验证通过/偏离/状态变更）。通过用绿色，偏离用红色，附证据摘要。人类一眼看到"什么时候发生了什么"。

### 对比表

**适合**：关键决策的 trade-offs、做什么 vs 不做什么

两列或三列对比，每列一个选项或方向。突出差异点。

### 卡片网格

**适合**：核心信念、不可妥协的价值、意图叙事

每张卡片一个要点，标题 + 一句话说明。卡片大小一致，网格排列。不要在卡片里放长段落——超过 3 行就太长。

### 清单

**适合**：反模式、不做什么、决策原则

每条一行，附理由（一句话）。反模式用视觉警告样式（红色边框、✕ 标记）。

### 进度条

**适合**：整体完成度

一条横条，填充比例 = completed/total。右侧显示百分比。填充色用绿色。

### 状态徽章

**适合**：Intent 状态标记

小标签，颜色 + 图标 + 文字：
- 完成 = 绿色 + ✓
- 进行中 = 黄色 + ◐
- 待执行 = 灰色 + ○
- 阻塞 = 红色 + ✕
- 需复审 = 蓝色 + ?

不要只靠颜色——加图标和文字（色盲友好）。

## 二、页面结构

### 顶部：一屏概览

像 PPT 封面 + 第一页。人类打开第一眼看到：

- 项目名 + 版本号
- 一句话说明项目在做什么（从北极星提取）
- 进度仪表盘（大数字：完成 X/Y、进行中、阻塞）
- 进度条

### Intent Map 区域

像 PPT 的核心页。项目推进到哪了：

- **SVG 依赖图**：一眼看出依赖链和阻塞点
- **状态分布**：仪表盘形式（待执行 X、进行中 Y、完成 Z、阻塞 W）
- **Intent 卡片网格**：每个 Intent 一张卡片，默认折叠，只显示 ID + 状态徽章 + 一句话叙事。点击展开显示详情

### 哲学区域

像 PPT 的"我们信什么"页。拆解成：

- **北极星**：一句话，全页最醒目
- **核心信念**：3-5 张卡片，每张一个关键句
- **不可妥协的价值**：清单，每条一句话 + 一句话理由
- **反模式**：清单，视觉警告样式
- **决策原则**：清单，遇到冲突时怎么取舍

### 愿景区域

像 PPT 的"我们要去哪"页。拆解成：

- **北极星**：和哲学呼应
- **问题空间**：一段话，用户在什么场景下遇到什么痛苦
- **意图叙事**：卡片网格，每个意图一张卡片，标题 + 一句话"为什么存在"
- **不做什么**：清单，和"做什么"同样醒目

### 架构区域（如果有）

像 PPT 的"怎么实现"页。拆解成：

- 关键决策对比表
- trade-offs 清单

### 验证历史区域（如果有）

像 PPT 的"质量怎么样"页。拆解成：

- **时间轴**：每次验证一个节点，通过/偏离用颜色区分
- **偏离清单**：哪些 Intent 偏离了，证据是什么

### 底部

- 提醒：这是只读投影，修改请编辑源文件后运行 `loom preview` 重新生成
- 可选：一个"复制项目状态"按钮

## 三、符合人类直觉

### 视觉权重

最重要的信息最大、最醒目。次要信息小、灰、退后：
- 北极星 → 全页最大字号
- 进度数字 → 大数字 + 颜色
- Intent ID → 中等字号，等宽字体
- 验收契约正文 → 小字，可展开才显示
- 辅助说明 → 最小字，灰色

不要所有文字一样大。人类靠大小判断重要性。

### 渐进披露

不要一次把所有信息摊开。先给摘要，想看详情再展开：
- Intent 卡片默认折叠，只显示 ID + 状态 + 一句话
- 点击展开后显示：意图叙事、验收契约、哲学锚点、验证历史
- 哲学的"决策原则"默认只显示标题，点击展开具体原则

### 信息分组

相关信息放一起，不相关的用空间分隔：
- 进度概览自成一个区块
- Intent 依赖图 + 详情面板自成一个区块
- 哲学、愿景、架构各自独立区块
- 区块之间用足够留白分隔（32-48px），不要挤

## 四、交互设计

### Tab 切换

用 tab 组织大板块（概览 / Intent Map / 哲学 / 愿景 / 架构）。
- 所有 tab 内容先渲染出来（堆叠），再用 JS 切换显示
- 选中的 tab 项加视觉标记（底部边框或背景色）
- 切换时内容区淡入，不要硬切
- 默认显示第一个 tab

### 点击展开

Intent 卡片、哲学章节、愿景叙事都用可折叠面板。
- 用 `max-height` 过渡，不要 `display: none`（后者无法动画）
- 折叠时 `max-height: 0; overflow: hidden`
- 展开时 `max-height: 2000px`（足够大）
- `transition: max-height 0.3s ease`
- 折叠状态有视觉提示（▶ / ▼）

### SVG 节点交互

- 点击节点 → 高亮该节点（加描边）+ 更新详情面板
- 其他节点取消高亮
- 详情面板内容用 `innerHTML` 更新

### 过滤/排序（如果 Intent 多）

Intent 超过 10 个时，加过滤和排序：
- 按状态过滤（全部 / 待执行 / 进行中 / 完成 / 阻塞）
- 输入框模糊搜索

## 五、视觉设计原则

参考 Anthropic 和 CopilotKit 的 Generative UI 设计系统：

### 平面克制

- **无渐变**：纯色背景
- **无阴影**：用边框区分层次
- **无模糊**：不要 filter: blur
- **无发光**：不要 text-shadow 或 glow
- **无装饰性 emoji**：用 CSS 形状或 SVG 路径做图标

### 字体克制

- 只用两种字重：400（正文）和 500（标题/强调）
- 不要 600、700——太重，破坏平面感
- 标题用句首大写，不要全大写
- 正文不要句中加粗，用 `code style` 标记实体名
- 最小字号 12px

### 边框和间距

- 边框 `1px solid`
- 卡片圆角 8-12px
- 卡片内边距 16-20px
- 卡片之间间距 16-24px
- 区块之间间距 32-48px——大间距区分大板块

### CSS 变量

```css
:root {
  --bg: ...;
  --surface: ...;
  --border: ...;
  --text: ...;
  --text-muted: ...;
  --accent: ...;
  --success: ...;
  --warning: ...;
  --danger: ...;
}
```

不要在 HTML 里硬编码颜色。所有颜色引用变量。

### 配色自定

不要求暗色也不要求亮色。选你觉得让信息最易读的方案。关键是：
- 背景和文字对比度足够（WCAG AA 标准）
- 状态颜色区分明显
- 不要花哨——信息是主角，视觉是配角

## 六、技术约束

- **单文件**：所有 HTML/CSS/JS 内联，零外部依赖
- **零构建**：双击即可在浏览器打开
- **内联 SVG**：依赖图直接写 `<svg>`，不用 `<img>`
- **语义化 HTML**：用 `<section>` `<nav>` `<article>` `<button>`
- **响应式**：手机宽度下布局不崩（`@media (max-width: 768px)`）

## 七、质量检查清单

生成前逐项确认：

**拆解质量**
- [ ] 每个信息块能回答一个明确的问题
- [ ] 没有块需要读超过 3 行文字才能理解
- [ ] 信息是拆解提炼的，不是 md 内容的搬运
- [ ] 选了合适的视觉形式（不是什么都用卡片或表格）

**功能**
- [ ] 双击能在浏览器打开，不需要任何外部资源
- [ ] SVG 依赖图节点颜色按状态编码
- [ ] 点击 Intent 能展开详情
- [ ] Tab 切换正常

**视觉**
- [ ] 无渐变、无阴影、无模糊、无发光
- [ ] 字重只有 400 和 500
- [ ] 所有颜色用 CSS 变量
- [ ] 边框统一 1px
- [ ] 信息层次清晰（大字重要，小字次要）

**可访问性**
- [ ] `@media (prefers-reduced-motion: reduce)` 禁用动画
- [ ] 对比度足够
- [ ] 点击区域 ≥ 44px
- [ ] 不仅靠颜色传达信息（颜色 + 图标 + 文字）

**响应式**
- [ ] 手机宽度下布局不崩
- [ ] SVG 依赖图横向滚动
- [ ] Tab 导航横向滚动
