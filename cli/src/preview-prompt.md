# LOOM Preview 生成提示词

你正在为 LOOM 项目生成一份 **HTML 可视化预览**。这份 HTML 是人类用户理解项目全局的唯一窗口——哲学、愿景、架构、Intent 进度，全部在一个页面里。

## 为什么用 HTML 而不是 Markdown

参考 Anthropic Claude Code 团队的实践：当信息密度超过 100 行 Markdown 就没人读了。HTML 能做到 Markdown 做不到的事——SVG 依赖图、tabs 切换、点击展开、颜色编码、响应式布局。你要充分利用这些能力，让这份预览**让人愿意打开、打开就能看懂、看懂就能决策**。

## 一、视觉系统

### 配色（暗色主题，参考 GitHub Dark）

| 用途 | 色值 |
|---|---|
| 页面背景 | `#0d1117` |
| 卡片背景 | `#161b22` |
| 卡片悬浮 | `#1c2128` |
| 边框 | `#30363d` |
| 主文字 | `#c9d1d9` |
| 次要文字 | `#8b949e` |
| 链接/标题强调 | `#58a6ff` |
| 完成 | `#3fb950` |
| 进行中 | `#d29922` |
| 阻塞 | `#f85149` |
| 待执行 | `#8b949e` |
| 需复审 | `#bc8cff` |

### 字体

```css
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'Noto Sans SC', Helvetica, Arial, sans-serif;
```

代码用 `ui-monospace, 'SF Mono', 'Cascadia Code', 'Source Code Pro', monospace`。

### 排版节奏

- 页面最大宽度 `1100px`，居中，左右 `2rem` 内边距
- 卡片之间间距 `1.5rem`，卡片内边距 `1.75rem`
- 标题层级：h1 `1.75rem` / h2 `1.35rem` / h3 `1.1rem` / 正文 `0.95rem` / 辅助文字 `0.8rem`
- 行高 `1.65`，段落间距 `0.75rem`
- **大量留白**——不要把内容挤在一起，呼吸感比信息密度重要

### 微交互

- 卡片 hover 时背景从 `#161b22` 过渡到 `#1c2128`（`transition: 0.2s`）
- 可点击元素 hover 时加 `cursor: pointer` + 轻微 `transform: translateY(-1px)`
- 折叠/展开用 `max-height` 过渡动画，不要瞬间出现/消失
- 进度条填充用 `transition: width 0.6s ease`

## 二、页面结构

### 1. 顶部 Hero 区

```
┌─────────────────────────────────────────────┐
│  LOOM 项目预览          [v1] badge          │
│  哲学驱动开发 · 哲学 → 愿景 → 架构 → Loop   │
│                                             │
│  ████████████░░░░░░░░░░  3/5 完成 (60%)     │
│                                             │
│  [总数 5] [完成 3] [进行 1] [待执行 1] [阻塞 0] │
└─────────────────────────────────────────────┘
```

- 标题 + 版本 badge + 一句话副标题
- 进度条：宽度撑满，圆角，绿色填充，右侧显示 `completed/total (百分比)`
- 统计卡片：横向排列，每个卡片一个数字 + 标签，数字用对应状态颜色

### 2. Tab 导航

固定在顶部下方，点击切换内容区域。Tab 项：

- **Intent Map**（默认选中）
- **哲学**
- **愿景**
- **架构**（如果没有架构数据则隐藏这个 tab）

Tab 切换时内容区淡入（`opacity` 过渡），不要硬切。

### 3. Intent Map Tab

这是最重要的 tab，分两部分：

#### 3a. SVG 依赖图（上方）

- 节点是 Intent，圆形或圆角矩形，内部显示 ID（如 `INT-001`）
- 边是 `depends_on` 关系，箭头从依赖指向被依赖
- **节点颜色按状态编码**（用上面的配色）
- 拓扑排序布局：无依赖的在最左，依赖链向右延伸。用 `topo_order` 数据定位
- 节点可点击。点击后：
  - 节点高亮（加描边 `stroke: #58a6ff`，`stroke-width: 3`）
  - 下方详情面板更新内容
- SVG 宽度自适应容器，高度根据节点数量动态计算（每层 `120px` 间距）
- 如果 Intent Map 还是模板（没有真实 Intent），显示提示："Intent Map 等待 Architect 设计"

#### 3b. Intent 详情面板（下方，点击节点后显示）

点击某个 Intent 节点后，这里显示该 Intent 的完整信息：

- **ID + 状态徽章**
- **意图叙事**（从 `narrative_ref` 解析，或显示 `narrative_ref` 引用）
- **验收契约**（`acceptance` 字段）
- **哲学锚点**（`philosophy_anchors` 列表，每个显示为可点击的 tag）
- **依赖**（`depends_on` 列表）
- **验证历史**（如果 `verifications` 中有该 Intent 的记录，显示每轮的 verdict + 证据）

默认显示第一个 Intent 的详情（或 `topo_order[0]`）。

#### 3c. Intent 表格（最下方）

完整列表，列：ID / 状态徽章 / 依赖 / 验收契约（截断到 80 字符 + `...`）。
行可点击，点击后同步更新详情面板 + 高亮 SVG 节点。

### 4. 哲学 Tab

- 如果有多个哲学文档，用**二级 tab** 切换
- 内容渲染：标题层级（h1/h2/h3）、粗体、列表、引用块（左侧蓝色边框）、行内代码、分隔线
- `<!-- LOOM_TEMPLATE -->` 标记的文档显示一个黄色提示条："这是模板，等待 Weaver 织造"
- 底线内化声明的 checkbox 渲染为可视觉勾选状态（`☑` 或 `☐`）

### 5. 愿景 Tab

- **北极星**高亮显示——大字号、蓝色、居中或左对齐突出
- 意图叙事列表：每个叙事是一个可折叠卡片
  - 折叠时显示标题（如 `INT-001：快速记录`）
  - 展开后显示完整叙事文本
  - 默认全部折叠
- `<!-- LOOM_TEMPLATE -->` 标记的愿景显示提示条

### 6. 架构 Tab（如果有数据）

- 渲染架构文档内容
- 如果没有 `architecture` 数据，这个 tab 不显示

### 7. 底部操作区（所有 tab 下方）

```
┌─────────────────────────────────────────────┐
│  [📋 复制项目状态]  [🔄 重新生成预览]        │
│                                             │
│  这是只读投影。修改请编辑 .loom/v1/ 下的源  │
│  文件，然后运行 loom preview 重新生成。     │
└─────────────────────────────────────────────┘
```

- **复制项目状态**按钮：点击后复制一段自然语言到剪贴板，格式如：
  ```
  LOOM 项目 v1 当前状态：
  - 进度：3/5 完成（60%）
  - 进行中：INT-004
  - 阻塞：无
  - 待执行：INT-005
  
  请基于此状态继续推进。
  ```
  复制后按钮文字短暂变为"已复制 ✓"（1.5 秒后恢复）
- **重新生成预览**按钮：显示提示"改完 md 后运行 `loom preview` 重新生成"
- 底部小字说明这是只读投影

## 三、交互细节

### 折叠/展开

```javascript
// 用 max-height 过渡，不要 display: none
// 折叠时 max-height: 0; overflow: hidden
// 展开时 max-height: 2000px（足够大）
// transition: max-height 0.3s ease
```

### Tab 切换

- 点击 tab 项时，所有 tab 内容 `display: none`，选中的 `display: block`
- 选中的 tab 项加底部蓝色边框（`border-bottom: 2px solid #58a6ff`）
- 切换时内容区 `opacity` 从 0 到 1 过渡（`transition: 0.2s`）

### SVG 节点点击

- 点击节点时：所有节点去掉高亮描边，被点击的节点加 `stroke: #58a6ff, stroke-width: 3`
- 详情面板内容用 `innerHTML` 更新
- 详情面板如果有滚动条，滚动到顶部

### 响应式

- `@media (max-width: 768px)`：
  - 统计卡片改为 2 列网格
  - SVG 依赖图横向滚动（`overflow-x: auto`）
  - Intent 表格隐藏验收契约列（太窄显示不下）
  - tab 导航改为横向滚动

## 四、技术约束

- **单文件**：所有 HTML/CSS/JS 内联在一个 `.html` 文件里
- **零依赖**：不用任何 CDN、外部库、外部字体、外部图片
- **零构建**：双击即可在浏览器打开，不需要 server
- **内联 SVG**：依赖图直接写 `<svg>` 标签，不用 `<img>`
- **语义化 HTML**：用 `<section>` `<nav>` `<article>` 等标签
- **可访问性**：按钮有 `aria-label`，tab 有 `role="tab"` 和 `aria-selected`
